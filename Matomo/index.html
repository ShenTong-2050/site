简介

Matomo 是领先的 Google Analytics 开放替代方案，让您可以完全控制您的数据。Matomo 让您可以轻松地从网站和应用程序收集数据，并将这些数据可视化并提取见解。

配置要求

运行配置

  • Web 服务器，例如 Apache、Nginx、IIS、LiteSpeed 等。
  • Matomo 4.x 需要 PHP 7.2.5 或更高版本。Matomo 也完全适用于 PHP 8。（较旧的 Matomo 3.x 需要 PHP 5.5.9 或 PHP 7.x 版本）
  • MySQL 5.5 或更高版本，或 MariaDB（默认启用）PHP 扩展pdo和pdo_mysql，或 mysqli 扩展。
  • Matomo 可以在任何操作系统上运行，例如 Linux（Ubuntu、RedHat、CentOS、Raspberry Pi OS 等）、Windows、macOS Server 或 FreeBSD。

推荐配置

我们建议使用最新的 PHP 8.x 版本，因为它比以前的 PHP 版本更节省内存且速度更快。

我们建议为您的数据库使用MySQL 8+或MariaDB。

为了充分利用 Matomo，您还需要一些额外的 PHP 扩展，例如用于生成迷你图（小图）的PHP GD 扩展、统计电子邮件报告中的图表以及 Matomo 移动应用程序中的图表。建议安装的 PHP 扩展列表是：

sudo apt-get install php php-curl php-gd php-cli mysql-server php-mysql php-xml php-mbstring

优化 Matomo

Matomo 报告是实时生成的，因此您应该可以立即在 Matomo 仪表板中看到数据。
关于性能：当您的Matomo已设置好，可以跟踪和报告网站的流量，但是如果您发现Matomo速度较慢，则需要采取额外的步骤来设置 CLI归档。这将在以下位置生成报告 定期而不是按需提供背景信息。

  • 默认情况下 Matomo 是实时工作的。但是在跟踪大型网站时我们强烈建议关闭 实时报告，例如每 1 或 6 小时仅处理一次今天的报告，这样对用户界面的访问将仅读取预处理的报告

Mysql 用户要求

安装 Matomo 时，您需要指定 MySQL 用户名和密码。MySQL 用户必须有权在数据库中创建和更改表。

MySQL 用户应具有 SELECT、INSERT、UPDATE、DELETE、CREATE、INDEX、DROP、ALTER、CREATE TEMPORARY TABLES、LOCK TABLES、FILE 的权限。

注意：Matomo 不能与 MySQL 负载均衡服务（如 ProxySQL）一起工作，除非您可以确保给定已建立连接中的所有查询始终发送到同一数据库服务器。

推荐的服务器大小

要知道要使用多少 CPU 以及多少 RAM 和磁盘磁盘空间，最重要的因素是您每月将在 Matomo 中跟踪多少请求。请求包括页面浏览量、事件、下载量等。其他影响服务器容量规划的因素包括创建了多少网站，以及需要处理多少用户数据。

作为一般规则，当每月跟踪超过 100 万次用户操作/页面浏览/事件时，我们建议至少拥有两台服务器，一台用于数据库，一台用于处理所有 HTTPS 请求（跟踪、API、UI）和报表处理。

配置建议

访问量 服务器配置
跟踪每月 100,000 次或更少的页面浏览量 一台服务器足以同时托管数据库和应用程序服务器，最低推荐配置：2 个 CPU、2 GB RAM、50GB SSD 磁盘。
每月跟踪 100 万次或更少的页面浏览量 一台服务器足以同时托管数据库和应用程序服务器，最低配置推荐：4 CPU、8 GB RAM、250GB SSD 磁盘。
每月跟踪 1000 万次或更少的页面浏览量 推荐两台服务器，1 个应用服务器，至少 8 个 CPU，16 GB RAM，100GB SSD 磁盘 或者 2 个应用服务器，至少 4 个 CPU，4 GB RAM，100GB SSD 磁盘，1 x 数据库服务器，至少 8 个 CPU，16 GB RAM，400GB SSD 磁盘。
每月跟踪 1 亿次或更少的页面浏览量 建议至少三台服务器 1
每月跟踪超过 1 亿的页面浏览量 至少五台服务器2

Matomo 每月可以支持超过 10 亿次页面浏览（通过设置多个 Matomo 实例的集群甚至更多）

如果您没有启用 PHP 7 的服务器或 MySQL/MariaDB 数据库，或者您不想在自己的服务器上托管 Matomo，我们已经创建了我们自己的 高质量 Matomo 托管服务：快速、可扩展、完全预装安全的 Matomo 服务，收取少量费用。我们的团队会为您处理所有复杂的事情，我们甚至可以迁移任何现有的 Matomo 数据

Note：不要丢失这些信息；这是您首次登录 Matomo 的唯一方式。每个 Matomo 安装中只有一个超级用户。超级用户可以执行管理任务，例如添加要监控的新网站、添加用户、更改用户权限以及启用和禁用插件。
