模板引用

ref 属性 适用于 直接访问底层 DOM 元素 的 场景

ref 与 v-for 中的 key 类似【唯一】，它允许我们在一个特定的 DOM 元素 或 子组件 实例 被 挂载后 获得 对它的直接引用，比如 在 组件挂载时 将 焦点 设置到一个 input 元素上，或 在一个元素上 初始化 一个 第三方库。

访问模板引用

组件挂载结束后 引用 都会被 暴露在 this.$refs 之上：

<script>    
export default {
    mounted() {
        this.$refs.name.focus()
    }
}
</script>

<template>
    请输入名称：<input ref="name" />
</template>

注意：你只能在 组件挂载后 才能访问模板引用。

v-for 中的模板引用

export default {
    data() {
        return {
            list:[1,2,3]        
        }    
    },
    mounted() {
        console.log(this.$refs.vfor)    
    }
}

<template>
    <div>
        <h2>v-for之模板的引用</h2>
        <ul>
            <li v-for="item in list" ref="vfor">
                {{ item }}
            </li>
        </ul>
    </div>
</template>

函数模板引用

除了使用 字符串 值 做名字，ref 属性 还可以绑定 为 一个函数，会在每次组件更新时 被调用。

该函数第一个参数为 对 该元素的引用。

<input :ref="(el) => { /*将 el 赋值给 一个数据属性 或 ref 变量*/ } ">

注意这里需要使用动态的 :ref 绑定 才能够 传入一个函数，当绑定的元素被卸载时函数也会被调用一次，此时传给 对该元素引用的 将是一个 null

export default {
    data() {
        return {
            age:'',
        }    
    }，
    mounted() {
        console.log(this.age)    
    }
}

<template>
    <div>
        <h2>函数模板引用：</h2>
        请输入年龄：<input :ref="(age) => { this.age = age }">
    </div>
</template>

组件上的 ref

子组件上的模板引用：

如果一个 子组件 使用的是 选项式API，被引用的组件实例 和 该 子组件的 this 完全一致， 这意味着 父组件 对 子组件 的 每一个属性 和 方法 都有完全的访问权。 这使得 父组件 和 子组件 之间 创建紧密耦合 的 实现细节 变得很容易， 当然也因此 应该 只在 绝对需要时 才 使用 组件引用。 大多数情况下你应该先使用 标准的 props 和 emit 接口 来 实现 父子组件的交互

<script>
import Child from './Child.vue'

export default {
    components: {
        Child    
    },
    mounted() {
        // this.$refs.child 时 <Child /> 组件的实例
    }
}
</script>

<template>
    <Child ref="child" />
</template>

限制对 子组件 实例的访问

export default {
    expose: ['publicData','publicMethod'],
    data() {
        return {
            publicData:'foo',
            privateData:'bar'        
        }    
    },
    methods : {
        publicMethod() {
            /* ... */        
        },
        privateMethod() {
            /* ... */        
        }
    }
}

通过 expose 的使用，父组件 通过模板引用 访问到 子组件的实例 后，仅能访问到 publicData 和 publicMethod

组件

定义

官方 推荐使用 .vue 结尾的 单文件组件，其中包括 <script>、<template>、<style> 三部分

组件的命名方式推荐采用 pascalCase 方式命名

  • 全局注册
    • 在打包过程中，未使用到的组件 也会被 保留
    • 影响长期的维护
  • 局部注册
    • 局部注册 组件 仅在 当前的组件中 可用，无法在 子组件 中 可用
    • 依赖关系明确
<script>
export default {
    data() {
        return {
            count:0        
        }    
    }
}
</script>

<template>
    <h2> {{ count }}</h2>
    <button @click="count++">加1</button>
</template>

<style></style>

组件的使用

特征

  • 通过 import 导入
  • 若要在模板中使用：需要通过 components 选项 注册
  • 当重复使用组件时，每个组件都会创建一个新的实例
  • 组件的名称推荐使用类似 pascalCase 的标签名，在 编译的时候 Vue 会区分大小写
  • 局部注册的组件 在 后代组件中 不可用

监听事件

App.vue

<script>
export default {
  data() {
    return {
      title:'标题',
      prography:[{id:1,content:'今天'},{id:2,content:'我在想'},{id:3,content:'吃什么'}],
      postFontSize:1,
    }
  },
  components: {
    VProps
  }
}
</script>

<template>
    <main>
        <div :style="{fontSize:postFontSize+'em'}">
          <Vprops :title="title" :prography="prography" @change-font-size="postFontSize+=0.2" />
        </div>
    </main>
</template>

components/vprops.vue

<script>
export default {
    props:[
        'title',     // 文章标题
        'prography', // 文章段落，数组形式的
    ],
    emits:['change-font-size'] 
}
</script>

<template>
    <h2> {{ title }} </h2>
    <ul>
        <template v-for="item in prography" :id="item.id">
            <template v-if="item.id == 2">
                <li class="red-cont">{{ item.id }} - {{ item.content }}</li>
            </template>
            <template v-else-if="item.id == 3">
                <li class="green-font">{{ item.id }} - {{ item.content }}</li>
            </template>
            <template v-else>
                <li class="pink-font">{{ item.content }}</li>
            </template>
        </template>
        <button @click="$emit('change-font-size')">放大字体</button>
    </ul>
</template>

动态组件

App.vue

<script>
import VProps from './components/vprops/vprops.vue'
import VSlot from './components/vslot/vslot.vue'

export default {
    components {
        VProps,         // 组件间的传值 
        VSlot,          // 传 html 组件
    },
    data() {
        return {
            tabs: ['VProps','VSlot'],
            slotMsg: '这是父组件中的内容',
            title:'冯齐的忏悔',  // 文章标题
            prography:[{id:1,content:'冯齐'},{id:2,content:'陈研'},{id:3,content:'相差十岁的爱情'}],
            fontSize:1,         // 放大字体
        }    
    }
}
</script>

<template>
    <div class="demo">
        <!--导航栏-->
        <button
        v-for="tab in tabs"
        :key="tab"
        :class="['tab-button', { active: currentTab === tab}]"
        @click="currentTab = tab"
        >
            {{ tab }}
        </button>
    
        <KeepAlive :include="['VProps']" :max="10">
            <!-- VSlot 组件 -->
            <template v-if="currentTab === 'VSlot' ">
              <component :is="currentTab" > {{ msg }} </component>
            </template>
            <!-- VProps 组件 -->
            <template v-else>
              <div :style="{fontSize:postFontSize+'em'}">
                <component :is="currentTab" :title="title" :prography="prography" @change-font-size="postFontSize+=0.2"  />
              </div>
            </template>
        </KeepAlive> 
    </div>
</template>

DOM 内模板解析注意事项

如果想在 DOM 中 直接书写 Vue 模板，Vue 则 必须从 DOM 中获取模板字符串，由于 浏览器 的 原生 HTML 解析行为限制，有一些注意事项：

这些 注意事项 仅限于 直接在 DOM 中编写模板的情况，如果你使用 以下 来源的 字符串 模板，就不需要关注这些注意事项了：

  • 单文件组件
  • 内联模板字符串（例如：template:....）
  • <script type="text/x-template">

大小写区分

HTML 标签 和 属性名称 是 不区分大小写的，所以浏览器会把 任何 大写的字符 解释为 小写。

这意味着当你使用 DOM 内的模板时，无论是 PascalCase【首字母大写】 形式的组件名称、cameCase 【驼峰】形式的 prop 名称，还是 v-on 的事件名称，都需要转换为等价的 kebab-case（短横线连字符）形式：

在 JavaScript 中 定义一个 DOM 模板

const BlogPost = {
    props: ['postTitle'],
    emits: ['updatePost'],
    template:`
        <h3> {{ postTitle }} </h3>
    `
}

转换为对应的 HTML 内容为：

<blog-post post-title="hello!" @update-post="onUpdatePost"></blog-post>

闭合标签

// 在 vue 模板解析器中 支持任意标签使用  `/>` 作为 闭合标志 例如：
<MyComponent />

// 在 html 中只有 例如 `input`、`img` 等少数标签 支持，所以如果你省略了关闭标签 则 意味着 标签没有结束
<my-component />
<span> hello </span>

// 解析为
<my-component>
    <span>hello</span>
</my-component>

// 所以正确写法为
<my-component></my-component>

元素位置限制

对于 在 table、ul、select 这些 带有 限制性标签 的 元素 中 使用 组件时 引发的问题 可以通过 is 属性解决，例如：

// 错误示例
<table>
    <blog-post-row></blog-post-row>
</table>

// 正确示例
<table>
    <tr is="vueLblog-post-row"></tr>
</table>

使用 is 属性 时 前边 必须加上 vue: 才能 被 正确解析，这是为了避免 和 原生的 自定义内置元素 混淆
