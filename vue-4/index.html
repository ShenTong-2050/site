生命周期

每个 Vue 组件实例 在 创建时 都需要经理一系列的初始化步骤，比如 设置好 数据侦听、编译模板、挂载实例到DOM，在这个过程中 它 也会运行被称为 生命周期 钩子的函数，让开发者有机会 在 特定阶段 运行自己的代码。

注册周期钩子

举例来说，mounted 钩子 可以 在组件 完成初始渲染 并 创建DOM 节点后 运行代码：

export default {
    mounted() {
        console.log(`the component is now mounted.`)    
    }
}

还有其他钩子：mounted、updated 和 unmounted

所有 生命周期 钩子函数 的 this 上下文 都会 自动指向 当前调用它的组件实例。【避免使用箭头函数定义生命周期钩子】

侦听器

基本示例

watch 选项可以在 每次 响应式 属性 发生变化 时 触发一个函数：

<script>
export default {
    data() {
        return {
            loading:false,
            question:'',
            answer:'default answer',
            profile : {
                name:'',
                age:'',
            }
        }
    },
    watch: {
        question(newQuestion,oldQuestion) {
            if (newQuestion.includes('?')) {
                this.getAnswer()
            }
        },
        // 监听 对象 某个属性的 更改
        // 'profile.name'(newName,oldName) {
        //     this.getName()
        // },
        // 'profile.age'(newAge,oldAge) {
        //     this.getAge()
        // }
        // 监听整个对象
        profile: {
            handler(newValue,oldValue) {
                this.editProfile(newValue,oldValue)
            },
            deep:true
        }
    },
    methods: {
        async getAnswer() {
            this.loading = true
            this.answer = 'Thinking...'
            try {
                const res = await fetch('https://www.demo.com/api')
                this.answer = (await res.json()).answer
            } catch(error) {
                this.answer = 'Error! Could not reach the API. ' + error
            } finally {
                this.loading = false
            }
        },
        async getName(newName,oldName) {
            try {
                const url = 'http://www.test.com/api/profile/edit?name='+this.profile.name;
                const res = await fetch(url).then(response => response.json())
                this.profile.name = res.data.name
                console.log(res.data.name)
            } catch(error) {
                this.name = 'Request name api something errror'
            }
        },
        async getAge(newAge,oldAge) {
            try {
                const url = 'http://www.test.com/api/profile/edit?age='+this.profile.age
                const res = await fetch(url).then(response => response.json())
                this.profile.age = res.data.age
                console.log(res.data.age)
            } catch(error) {
                this.name = 'Request age api something errror'
            }
        },
        async editProfile(newValue,oldValue) {
            const url = 'http://www.test.com/api/profile/edit?name='+newValue.name+'&age='+newValue.age
            const res = await fetch(url).then(response => response.json())
            console.log(res.code)
        }
    }
}
</script>

<template>
    <div>
        <h2>默认浅层 watch 监听：</h2>
        <p>
            问个问题：<input type="text" v-model="question" :disabled="loading" />
        </p>
        <p> {{ answer }} </p>
    </div>

    <div>
        <h2>深层 watch 监听：</h2>
        <div>
            输入姓名：<input type="text" v-model="profile.name" />
        </div>
        <div>
            输入年龄：<input type="text" v-model="profile.age" />
        </div>
        <span>{{ profile.name }} 今年 {{ profile.age }} 岁!</span>
    </div>
</template>

1、侦听器 默认为 浅层侦听

2、当 对象 结构过于庞大 时 慎用 深层侦听器，因为这会导致 开销变大，建议只在必要情况下使用，并且要留意性能

及时回调侦听器

watch 默认是 惰性加载，只有 当 数据源变化 的 时候 才会执行回调，如果想在 创建 侦听器的时候 立即执行一遍 回调，可以通过如下这种方式进行：

<script>
export default {
    data() {
        return {
            immediate:''        
        }    
    },
    watch:{
        immediate: {
             handler(newText,oldText) {
                console.log(newText)
                console.log(oldText)
            },
            immediate:true
        }
    }
}
</script>

<template>
    <div>
        <h2>及时回调侦听：</h2>
        获取最新并修改结果：<input type="text" v-model="immediate" />
    </div>
</template>

一次性侦听

顾名思义，一次侦听 就是 在 数据源变化时 触发一次 更新，通过 增加 once:true 对象 实现

export default {
    data() {
        return {
            nickname:''        
        }    
    },
    watch:{
        nickname:{
            handler(newNickname,oldNickname) {
                console.log(newNickname)
                console.log(oldNickname)            
            },
            once:true
        }    
    }
}

<template>
    <div>
        <h2>一次性侦听：</h2>
        一个月只能修改一次的昵称：<input type="text" v-model="nickname" />
    </div>
</template>

回调的触发时机

  • 响应式状态 变化：如 data 或 computed 中 数据的修改
  • 组件更新 和 侦听器 回调函数 的 触发：
    • 组件更新：响应式 数据 的 变化 会 触发 vue 组件的 重新渲染
    • 侦听器 回调：watch 监听的回调函数
  • 批量处理以避免重复调用：为了提高性能避免重复调用，Vue 会 将 组件更新 和 侦听器回调 进行 批量处理，例如：如果你在一次 同步操作中 对 数组 进行了多次修改，Vue 不会立即触发多次回调，而是会进行批量处理，只触发一次更新
  • 回调的触发顺序：默认情况下 侦听器回调 会在 父组件（如果存在）更新之后、所属组件更新之前 被 调用，这意味着 在 父组件更新之后 和 当前组件更新之前 回调函数被 调用
  • DOM状态：由于 侦听器 回调 在 DOM 更新之前 被 调用，因此 在 回调中 访问 DOM 时 DOM 仍然处于 更新前的状态。如果你需要在侦听器回调中访问更新后的 DOM，可以使用 this.$nextTick 来 确保在 DOM 更新完成之后进行访问。
export default {
    data() {
        return {
            timetick : ''        
        }    
    },
    watch:{
        timetick(newValue,oldValue)  {
            console.log('Message changed from',oldValue,'to',newValue)

            // 尝试访问 DOM
            // console.log(this.$el)
            console.log('DOM befor update: ',document.getElementById('timetick').textContent);

            // 使用 $nextTick 来 确保 DOM 已更新
            this.$nextTick(() => {
                console.log('DOM after update: ',document.getElementById('timetick').textContent);
            })
        }
    },
    methods: {
        changeMessge() {
            this.timetick = 'newTimeTick'
        }    
    }
}

<template>
    <div>
        <h2>回调的触发时机：</h2>
        <span>{{ timetick }}</span>
        <button @click="changeMessge" >Test Time Tick</button>
    </div>
</template>

总结：

1、由于 Vue 的 响应式系统 和 DOM更新 是 高度优化 和 同步的，但在 复杂 或 大量数据 变动的场景下，使用 $nextTick 能确保操作是在DOM 完全更新后进行的。

2、尽管 watch 回调通常能够直接访问更新后的 DOM，但为了代码的稳定性 和 可维护性，依然建议在需要确保 DOM 更新后的操作中 使用 $nextTick。

同步侦听器

同步侦听器 指的是 立即响应数据变化 的 侦听器。通常 vue 的 watch 侦听器 是在 下一次 DOM 更新周期之前 执行的，确保 数据变化 和 DOM 更新之间的顺序。 然而有时 你可能希望在数据变化时立即做出相应，而不必等待 DOM更新周期完成。这就涉及到所谓的同步侦听器。

export default {
    data() {
        profile:{
            name:'',
            age:''        
        }
    },
    watch:{
        profile:{
            handler() {
                            
            },
            flush:'sync'
        }
    }
}

简单来说 同步侦听器 就是在 响应数据 发生变化后 立即做出反应，也就是会在 vue进行任何更新之前触发

this.$watch()

除了 生命周期中的 watch 关键字为，还可以通过 $watch() 方法 来 命令式的 创建一个 侦听器：

export default {
    created() {
        this.$watch('question',(newQuestion) => {
            // ....  
            console.log(newQuestion)      
        })    
    }
}

<template>
    <div>
        <h2>默认浅层 watch 监听：</h2>
        <p>
            问个问题：<input type="text" v-model="question" :disabled="loading" />
        </p>
        <p> {{ answer }} </p>
    </div>
</template>

适用于 特定条件下的 侦听器，例如 只 侦听 响应 用户交互的内容，除此之外 还允许 你 提前终止 该 侦听器

停止侦听器

大多数场景中，用 watch 与 $watch() 实例方法 声明 的 侦听器 会在 宿主组件 卸载时 自动停止，所以我们无需关心怎么停止它。

在 少数场景中，我们需要在 组件写在之前 就 停止一个侦听器，这时 可以调用 $watch() API 返回的函数：

<script>
export default {
    data() {
        return {
            question: '',            // 初始化 question 字段
            unwatchQuestion: null,   // 存储 question 字段的 监听函数
        }    
    },
    created() {
        // 注册 question 字段的监听事件
        this.unwatchQuestion = this.$watch('question',(newQuestion) => {
            console.log(newQuestion)
        })    
    },
    methods : {
        stopQuestionWatch() {
            if (this.unwatchQuestion) {
                this.unwatchQuestion() // 调用 取消 侦听器 函数
            }
        }    
    }
}
</script>

<template>
    <div>
        <h2>默认浅层 watch 监听：</h2>
        <p>
            问个问题：<input type="text" v-model="question" :disabled="loading" />
        </p>
        <p> {{ answer }} </p>
        <div><button @click="stopQuestionWatch()">停止Question侦听函数</button></div>
    </div>
</template>
