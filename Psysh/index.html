关于Psysh

Psysh 是一个 PHP 的 REPL（Read-Eval-Print Loop）工具，它提供了一个交互式的命令行环境，让您可以在命令行下输入 PHP 代码片段并立即看到其执行结果。Psysh 具有丰富的特性，包括语法高亮、自动完成、历史记录、查看和编辑变量值、显示方法和属性、调试工具等， 使得在命令行下进行 PHP 代码调试、验证和测试变得更加方便和快捷。【Psysh手册】

Psysh 可以用作一个独立的 REPL 工具，也可以嵌入到您的 PHP 项目中，作为一个开发工具来进行交互式调试和测试。Psysh 支持 PHP 5.4 及以上版本，并且可以与多种命令行终端配合使用。您可以通过 Composer 安装 Psysh，然后在命令行下运行 “psysh” 命令来启动 Psysh。

REPL 是 Read-Eval-Print Loop（读取-求值-输出循环）的缩写。它是一种计算机编程环境，通常用于解释性编程语言，提供了一种交互式的编程方式。 在一个 REPL 环境中，用户可以输入一条代码或表达式，然后系统会立即对其进行求值并返回结果，这样用户可以实时看到代码的执行结果。用户还可以继续输入更多的代码或表达式，形成一个循环的交互过程。 REPL 提供了一种便捷的方式来进行代码测试、验证和调试。用户可以在 REPL 环境中快速尝试不同的代码片段，观察其执行结果，从而加快开发和调试的速度。REPL 环境通常包括一个命令行界面或交互式界面，如 Python 的交互式解释器、Ruby 的 irb、JavaScript 的 Node.js REPL 等。 REPL 环境在学习编程、探索编程语言特性、进行快速原型开发和进行交互式数据分析等场景中非常有用，并广泛应用于各种编程语言和开发工具中。

特性

  • 语法高亮：Psysh 可以对输入的 PHP 代码进行语法高亮显示，使代码更易读。
  • 自动完成：Psysh 支持自动完成，可以帮助您输入 PHP 代码时自动补全函数、类、变量等名称。
  • 历史记录：Psysh 会记录您在 REPL 中输入的代码，并允许您通过上下箭头键来访问以前输入的代码，方便查找和重用。
  • 查看和编辑变量值：Psysh 允许您查看和编辑当前的变量值，方便进行调试和验证。
  • 显示方法和属性：Psysh 可以显示对象的方法和属性列表，方便您在 REPL 中探索对象的结构和内容。
  • 调试工具：Psysh 提供了一些内置的调试工具，如 “wtf” 命令用于查看函数调用栈，“debug” 命令用于启动调试器等。

配置文件

配置项 作用
casters 这个配置项指定了使用 symfony/var-dumper 库进行输出时，如何对某些类型的变量进行格式化和显示。示例配置文件默认定义了一个自定义类型 MyFooClass 和对应的格式化函数MyFooClassCaster::castMyFooObject，表示在输出 MyFooClass 类型的变量时，会调用 MyFooClassCaster::castMyFooObject 函数来格式化输出。
colorMode 这个配置项指定了在终端中输出时，是否启用颜色。可选的值有：\Psy\Configuration::COLOR_MODE_FORCED 强制启用颜色、\Psy\Configuration::COLOR_MODE_DISABLED 禁用颜色、\Psy\Configuration::COLOR_MODE_AUTO 自动检测终端是否支持颜色。
commands 这个配置项定义了 PsySH Shell 中可用的命令。这里只定义了一个解析 PHP 代码的命令 \Psy\Command\ParseCommand。
defaultIncludes 这个配置项指定了在每个 PsySH 会话开始时需要自动加载的文件。这里定义了一个 bootstrap.php 文件，如果存在的话，就加载它
eraseDuplicates 这个配置项指定了历史记录是否允许出现重复的命令。如果设置为 true，则不会出现重复的命令，新的命令会覆盖旧的命令；如果设置为 false，则允许出现重复的命令。
errorLoggingLevel 这个配置项指定了记录错误日志的级别。只有符合这个级别的错误才会被记录下来。默认值是 E_ALL & ~E_NOTICE，表示记录所有错误，但不记录 NOTICE 级别的错误。
forceArrayIndexes 这个配置项指定了是否始终显示数组的索引，即使数组的键是数字。如果设置为 true，则始终显示数组的索引；如果设置为 false，则只有当数组的键不是数字时才会显示键。
historySize 这个配置项指定了历史记录最多可以包含的命令数量。如果设置为 0，则历史记录的数量是无限的。
interactiveMode 这个配置项指定了 PsySH 是否在交互模式下运行。可选的值有：\Psy\Configuration::INTERACTIVE_MODE_FORCED 强制交互模式、\Psy\Configuration::INTERACTIVE_MODE_DISABLED 禁用交互模式、\Psy\Configuration::INTERACTIVE_MODE_AUTO 根据输入的连接类型自动选择。
matchers 这个配置项定义了用于自动补全的匹配器。默认定义了两个用于自动补全 MongoDB 数据库和集合名称的匹配器。
pager 这个配置项指定了在输出超过屏幕内容时进行自动分页的分页器。默认使用 more，超出一屏时按下 空格键 或 回车键 显示下一页。
prompt 这个配置项指定了 PsySH Shell 中的提示符。
rawOutput 这个配置项指定了是否以 var_export 格式输出
historyFile 指定历史命令存储文件绝对路径

用法

  • help 命令
1

psysh
>help               // 获取支持的命令
>help ls            // 获取 ls 命令的用法示例

1）ls 命令用于列出给定对象（变量、对象、数组等）的属性、方法或数组元素。
2）show 命令用于显示各种信息，如类的继承关系、方法、属性、命名空间、函数等。
3）wtf【What the Function】命令是 Psysh 中的一个命令，用于显示最近的异常的堆栈跟踪信息。通常在调试过程中，程序会抛出异常，然后我们需要查看堆栈跟踪信息来帮助定位问题。这时就可以使用 wtf 命令来查看异常的详细信息，包括出错文件、代码行数以及堆栈跟踪信息等。例如，输入 wtf 命令可以显示最近一次的异常信息，而输入 wtf -a 命令可以显示完整的堆栈跟踪信息。在使用 wtf 命令时，还可以通过指定参数 incredulity 来控制显示的堆栈跟踪信息的行数。
4）whereami是Psysh的一个命令，它用于显示当前代码执行的位置。当你在调试代码时，如果你忘记了当前位置是在哪个文件或者类的哪个方法里，你可以使用这个命令来快速查看。
5）timeit 是一个命令，用于测量 PHP 代码执行的时间。它会运行一段给定的 PHP 代码并计算执行所花费的时间，然后返回运行该代码的时间。可以使用它来比较两个实现同一任务的算法之间的性能差异。

// Windows.php
interface Windows {
    public function render();
}

// SimpleWindow.php
class SimpleWindow implements Windows {
    public function render() {
        $returnString = <<<EOL
+-------------+
|             |
|             |
|             |
|             |
|             |
+-------------+
EOL;
        return $returnString . PHP_EOL;
    }
    
    public static function test() {
        return "hello world";
    }
}

// DecoratedWindow.php
class DecoratedWindow implements Windows {
    protected $windowReference;
    public function __construct(Windows $windowReference) {
        $this->setWindowReference($windowReference);
    }
    public function render() {
        return $this->getWindowReference()->render();
    }
    public function getWindowReference() {
        return $this->windowReference;
    }
    public function setWindowReference(Windows $windowReference) {
        $this->windowReference = $windowReference;
    }
}

// TitledWindow.php
class TitledWindow extends DecoratedWindow {
    public function render() {
        $string = $this->addTitle();
        return $string . parent::render();
    }
    protected function addTitle() {
        $returnString = <<<EOL
+-------------+
|Title        |
EOL;
        echo $returnString . PHP_EOL;
    }
}

// 命令行配置文件 command.php
return [
    'attribute' => 'app\\common\\command\\Attribute',

// 命令行处理类文件 Attribute.php
namespace app\common\command;

use think\console\Command;
use think\console\Input;
use think\console\Output;
use think\facade\Env;

class Attribute extends Command
{
    protected function configure() {
        $this->setName('attribute')
            ->setDescription('export attribute uuid');
    }

    protected function execute(Input $input,Output $output) {
        $file = Env::get('root_path') . 'public/static/file/attribute.csv';
        $handle = fopen($file,'r');
        $uuid = array();
        $data = '';
        while (!feof($handle)) {
            $data .= fread($handle,2048);
        }
        fclose($handle);
        $lineArr = preg_split('/(\r\n|\n|\r)/',$data);
        for ($i=0;$i<count($lineArr)-1;$i++) {
            $tmp = explode(',',$lineArr[$i]);
            array_push($uuid,"'" . $tmp[0] . "'");
        }
        $json = json_encode($uuid);
        file_put_contents('./public/static/file/attribute_uuid.txt',implode(',',$uuid));
        // 1.0 版本后将被弃用
        // \Psy\Shell::debug(get_defined_vars());
        \Psy\debug(get_defined_vars());

        echo "success count = " . count($uuid);
    }
}
