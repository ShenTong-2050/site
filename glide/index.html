关于图像处理

Glide

Glide 是一个非常易于使用的PHP编写的按需图像处理库。它通过 HTTP 提供了简单明了的 API，类似于像 Imgix 和 Cloudinary 这样的云图像处理服务。Glide 利用了强大的库，如 Intervention Image（用于图像处理和操作）和 Flysystem（用于文件系统抽象）。

特性

  • 使用简单的基于 HTTP 的 API 调整大小 和 添加图像效果。
  • 处理后的图像会自动缓存并通过响应标头进行提供。
  • 可以创建自己 的 图像处理服务 或 直接将 Glide 集成到应用程序中。
  • 支持 GD库 和 Imagick PHP扩展。
  • 支持多种响应方法，包括PSR-7、HttpFoundation等。
  • 能够使用HTTP签名保护图像URL。
  • 通过Flysystem库与许多不同的文件系统配合使用。
  • 由经过实战验证的Intervention Image图像处理和操作库提供支持。

HttpFoundation 是 Symfony 框架中的一个组件，它提供了一个面向对象的 API，用于处理 HTTP 请求和响应。它为开发人员提供了一个方便的方式来操作和处理与 HTTP 相关的数据，例如请求参数、头部信息、Cookie、会话等。

HttpFoundation 组件包含了一系列类和方法，可以帮助你构建和处理 HTTP 请求和响应。它提供了对请求方法、URL、查询参数、表单数据、文件上传等的访问和操作。同时，它还可以轻松地设置和获取响应状态码、头部信息、内容等。

通过使用 HttpFoundation 组件，开发人员可以方便地处理和操作 HTTP 请求和响应，包括验证和过滤输入数据、处理表单提交、设置响应头部、发送响应内容等。这样，开发人员可以更专注于应用程序的业务逻辑，而无需过多关注底层的 HTTP 协议细节

关于 Intervention Image

Intervention Image 是一个用于 PHP 的开源图像处理库，用于处理和操作图像。它提供了一系列功能，允许开发人员对图像进行各种操作，包括调整大小、裁剪、旋转、添加滤镜和水印等。

功能：

  • 调整大小和裁剪：可以轻松地改变图像的尺寸并裁剪图像以适应不同的需求。
  • 滤镜和效果：支持各种滤镜和特效，如模糊、锐化、颜色调整等，以改变图像的外观。
  • 水印：允许添加文本或图像水印到图像上，用于版权保护或品牌标识。
  • 旋转和翻转：可以旋转图像并在水平或垂直方向翻转图像。
  • 文件格式转换：支持不同的图像文件格式之间的转换，如JPEG、PNG、GIF等。
  • 图像信息获取：可以获取图像的各种信息，如宽度、高度、颜色深度等。

配置项 类型 含义
source string or object 指定原始图像文件的目录或源。Glide 将从该目录中读取原始图像文件，并在处理后将缓存的图像保存到不同的目录。
source_path_prefix string 为缓存文件名添加前缀，以防止与其他文件冲突。可以用于将缓存图像与其他文件隔离开来。
cache string or object 指定缓存处理后图像的目录。Glide 将处理的图像副本保存到这个目录中，以便以后的请求可以快速访问。
cache_path_prefix string 缓存目录 前缀
temp_dir string 用于存储缓存 EXIF 数据的临时目录（默认为 sys_get_temp_dir()）
base_url string 指定图像的基本 URL，用于生成完整的图像 URL。通常是你的网站域名或图像服务器的根路径。
driver string 指定用于图像处理的驱动程序。Glide 支持不同的图像处理库，如 GD、Imagick 和 Gmagick。你可以根据需要选择合适的驱动程序。
group_cache_in_folders bool 布尔值，用于确定是否将缓存的图像分组存储在子文件夹中。这可以帮助管理大量缓存图像的文件系统性能。
cache_path_prefix string 为缓存的图像文件添加前缀，以防止与其他文件冲突。可以用于将缓存图像与其他文件隔离开来。
cache_with_file_extensions bool 布尔值，用于确定是否在缓存文件名中包含原始图像文件的扩展名。这可以帮助在缓存中区分不同类型的图像。
max_image_size array 限制图像的最大宽度和高度，以防止恶意请求导致服务器负载过高。
defaults array 指定默认的图像处理选项，用于在没有指定特定选项的情况下处理图像。
presets array 定义一组预定义的图像处理选项，以便在处理图像时快速应用。这可以用于创建缩略图或不同尺寸的图像。
watermarks string or object 允许在图像上添加水印。你可以配置水印的位置、图像路径、透明度等属性。
watermarks_path_prefix string 水印文件系统路径前缀
response string or object 指定图像处理后的响应类型。可以是文件响应、流响应、Symfony 响应等，用于定义图像输出方式。
cache_with_query_string bool 布尔值，确定是否考虑查询字符串参数来生成缓存。这可以用于根据查询参数缓存不同版本的图像。
cache_path_callable closure 允许自定义确定图像缓存路径的回调函数。通过这个回调函数，你可以根据不同的逻辑来定义缓存路径。

<?php

class Glide extends Controller
{
    protected string $host              = 'http://www.mine.cd';
    protected string $source_path       = '';
    protected string $cache_path        = '';
    protected string $temp_path         = '';
    protected string $water_mark_path   = '';
    protected string $img_operate_route = '/img/operate';
    protected Server $server;
    protected array  $config;           // 图像 配置 数组信息
    // 签名信息
    private   string $sign_key          = 'shentongtong123'; 
    
    public function __construct() {
        $data = input();
        $this->config = !empty($data) ? $data : [];
    
        $staticPath = Env::get('root_path') . 'public/static/image';
        $this->source_path          =   $sourcePath = $staticPath;
        $this->cache_path           =   $staticPath . '/cache/';
        $this->temp_path            =   $staticPath . '/temp/';
        $this->water_mark_path      =   $staticPath . '/watermark/';
    
        $this->server = ServerFactory::create([
            // 'source'                        =>      $this->source_path,
            'source'                        =>      new Filesystem(new LocalFilesystemAdapter($this->source_path)),
            'source_path_prefix'            =>      '',
            'cache'                         =>      $this->cache_path,
            // 'cache'                         =>      new Filesystem(new LocalFilesystemAdapter('path/to/cache/folder')),
            'cache_path_prefix'             =>      '',
            'watermarks'                    =>      new Filesystem(new LocalFilesystemAdapter($this->source_path)),
            'watermarks_path_prefix'        =>      '',
            'temp_dir'                      =>      $this->temp_path,
            'group_cache_in_folders'        =>      false,       // 是否将缓存的图像分组存储在文件夹中
            'cache_with_file_extensions'    =>      true,        // 用于确定是否在缓存的图像文件名中包含文件扩展名
            'cache_path_callable'           =>      function ($img,$data) use($sourcePath) { // 通过动态函数返回根据自定义规则生成的缓存路径
                // $img => banner/logo.png 【相对于 source_path 的图片地址】
                // $data => array('w' => 100, 'bg' => fuchsia) 【图片处理的相关参数】
                // $sourcePath => D:\phpstudy\WWW\minea\public/static/image【图片处理的根目录】
                $mimeInfo = explode('/',getimagesize($sourcePath.'/'.$img)['mime']);
                $cachePath = explode('/',$img)[0];
                return $cachePath . '/' .md5($img.'/'.serialize($data)).'.'.$mimeInfo[1];
            },
            'driver'                        =>      'imagick',  // gd 或者 imagick
            'defaults'                      =>      [   // 请求中 不存在 对应参数 则 使用 默认配置
                'w'         =>      300,    //  默认宽度 300
                'h'         =>      200,    //  默认高度 200
                'fit'       =>      'crop', //  默认裁剪方式为裁剪
                'mark'      =>      'logo.png',
                'markw'     =>      30,
                'markpad'   =>      5,
            ],
            'presets'                       =>      [   // 通过 预设名称 重用一组 参数 避免重复代码
                'small' => [
                    'w' => 150,
                    'h' => 150,
                    'fit' => 'crop',
                ],
                'medium' => [
                    'w' => 800,
                    'h' => 600,
                    'fit' => 'crop',
                ],
            ],
            'base_url'                      =>  $this->host, 
            
            // SymfonyResponseFactory 将 处理后的图像 直接输出 到 浏览器
            // FileResponseFactory  保存文件
            // StreamResponseFactory 返回图像数据流
            // 默认 返回图像 url
            // 自定义 响应 CustomResponseFactory implements ResponseInterface
            // 'response'                      =>  '',
            
            'sign_key'                      =>  $this->sign_key,
            'max_image_size'                =>  500 * 500, // 限制图像最大为 250000 像素
        ]);
    }
}

安全

使用 HTTP 签名保护您的 Glide 图像服务器。通过使用私钥对每个请求进行签名，就无法对 URL 参数进行任何更改。

强烈建议您在生产环境中使用签名 URL，否则您的应用程序将容易受到大规模图像调整大小攻击。

public function create() {
    $data = $_GET;

    // 签名校验
    SignatureFactory::create($this->sign_key)->validateRequest($this->img_operate_route,$data);

    $img = $data['img'];
    unset($data['img']);

    $this->server->outputImage($img,$data);
}

public function avatar() {
    // 调整图像 方式一：返回 形如 http://mine.cd/img/operate?img=avatar%2Fcat.png&w=200&h=100&fit=crop&sign=82fbd4bd6a8ec80216ebb84396339af6 的地址
    $avatar = UrlBuilderFactory::create($this->host,$this->sign_key)->getUrl($this->img_operate_route,['img' => 'avatar/cat.png','w' => 200, 'h' => 100, 'fit' => 'crop']);
    // 调整图像 方式二：返回 形如 avatar/4215a7d76f5463cf723c0596693b03d2.jpeg 的 图片地址
    $cachePath = $this->server->makeImage($img,$data);
    $avatar = '/static/image/cache/' . $cachePath;
    // 调整图像 方式三：返回 base64 格式的图像数据
    $avatar = $this->server->getImageAsBase64('avatar/cat.png',['w' => 200, 'h' => 100, 'fit' => 'crop']);
    return $this->fetch('avatar',['avatar' => $avatar]);
}

public function resize() {
    $img = 'banner/grass.png';
    // 1) contain 默认，适用于 原图大 生成小图 的情况 宽高比不变
    $data = ['fit' => 'contain','w' => 500, 'h' => 500];
    // 2) max: 与 contain 相反，适用于 原图小 生成大图 的情况，宽高比不变
    // $data = ['fit' => 'max', 'w' => 500, 'h' => 500];
    $resize = $this->server->getImageAsBase64($img,$data);
    return $this->fetch('avatar',['avatar' => $resize]);
}


// 3) fill: 图像将被调整大小以适应指定的宽度和高度边界，而不会被裁剪或扭曲。
