Markdown 前世今生

Markdown 是一种轻量级的标记语言，它可以让你使用易读易写的纯文本格式编写文档，然后将其转换为 HTML 格式。

它由约翰·格鲁伯（John Gruber）和亚伦·斯沃茨（Aaron Swartz）在 2004 年创建，旨在让人们更容易地编写和阅读文档，而不需要使用复杂的 HTML 代码。

Markdown 语法简单明了，易于学习，可以快速地将文本转换为结构化的 HTML 文档。它在写作、博客、文档等方面都有广泛的应用。

支持编辑器：

  • VSCode / Atom
  • Vim / Emacs / Sublime Text / Notepad++
  • IntelliJ IDEA / Android Studio / WebStorm
  • Ulysses / Mou / Typora / Markpad

块级 HTML 元素”例如

、 、 
 、 
 等” 必须用空行与周围内容分隔，并且开头和结尾块的标签不应使用制表符或空格缩进。


因为 HTML 块级元素通常是独立的内容块，它们应该在视觉上与周围内容分开，以便更好地区分。在 Markdown 中，如果您在两个段落之间插入一个块级 HTML 元素，例如 
，则需要在元素前后使用一个空行分隔，以确保该元素被视为独立的内容块，而不是段落的一部分。如果没有使用空行分隔，则 Markdown 解析器可能会将该元素与前面或后面的段落合并，导致渲染错误。


此外，由于 Markdown 是基于文本的格式，因此在 Markdown 中使用制表符或空格缩进会导致 HTML 标签的格式不正确。如果在 HTML 块级元素的开头或结尾使用缩进，则可能会导致该元素的格式不正确，从而使渲染结果出现问题。因此，在 Markdown 中，应避免使用制表符或空格缩进来缩进 HTML 块级元素的开头或结尾。

Markdown Extra


Markdown Extra 与 Markdown 语法的主要区别在于 Markdown Extra 扩展了一些新的语法和功能，使得 Markdown 更加强大和灵活，但其不是官方规范的一部分。


支持的平台和工具：


    

  • GitHub：GitHub 支持部分 Markdown Extra 的语法，并且可以在 GitHub Pages 中使用该语法来编写网站内容
    • 

  • GitLab：GitLab 也支持部分 Markdown Extra 的语法，可以在 GitLab 中编写项目文档和 README 文件等
    • 

  • Jekyll：Jekyll 是一个静态网站生成器，支持使用 Markdown Extra 语法来编写网站内容
    • 

  • Hugo：Hugo 是另一个静态网站生成器，也支持使用 Markdown Extra 语法来编写网站内容
    • 

  • Typora：Typora 是一款支持实时预览的 Markdown 编辑器，支持 Markdown Extra 的语法
    • 





Jekyll 是一个静态网站生成器，它使用 Markdown、Liquid 模板语言和 YAML 前置数据来生成静态网站。Jekyll 最初是由 Tom Preston-Werner 创建的，他是 GitHub 的联合创始人之一。Jekyll 最初是作为 GitHub Pages 的默认静态网站生成器而开发的，后来成为了一个独立的开源项目。


Jekyll 基于 Ruby 编程语言开发，它使用 Liquid 模板语言来将模板和数据合并生成静态网页。Jekyll 支持 Markdown、Textile 和 HTML 等多种标记语言，并且可以使用 Front Matter 来添加页面的元数据。Jekyll 还提供了许多内置的插件和主题，方便用户进行定制和扩展。
使用 Jekyll 可以快速、方便地创建静态网站，并且可以将生成的网站部署到 GitHub Pages、Amazon S3、Netlify 等多个平台。


Jekyll 也被广泛应用于博客、文档、个人网站等多种场合。

脚注


在 Markdown Extra 中，脚注（Footnotes）是一种在文本中添加注释的方法。脚注可以帮助读者快速查看和理解文本中的特定术语或概念，而不会干扰到文本的流畅性


### 脚注

PHP是世界上最好的语言![^1]
[^1]: 关于php是什么这件事





1）脚注定义可以在文档中的任何位置找到，但脚注将始终按照它们在文本中链接的顺序列出

2）每个脚注必须有一个独特的名称。该名称将用于将脚注引用链接到脚注定义，但对脚注的编号没有影响。名称可以包含 HTML 中的id 属性内有效的任何字符

3）脚注内容一般都是在内容最下方

PHP Markdown 解析器


原始的 Markdown 实现已经演变为 CommonMark 和 GitHub-Flavored Markdown，它们提供了更好的代码块、表格和其他一些原始规范中没有的功能。

Parsedown【支持 Markdown Extra 语法】


Parsedown 是 PHP 中最受欢迎的 Markdown 库之一，由 Emanuil Rusev 编写。目前在 GitHub 上已获得超过 14,000 多个 star，并且在 Packagist 上迄今已经有超过 7500 万次下载。它被用于几个静态 CMS 软件、Laravel < 6 和其他超过 600 个项目（根据 Packagist 上的报告）。


Parsedown 支持大部分 CommonMark 语法，但并非全部。它是一个高度主观的库，有意地挑选出了 Markdown 语法中最常用的一些特性，并对不常用的特性开放扩展。




composer require erusev/parsedown

composer require erusev/parsedown-extra




// 解析 并将 Markdown 输出为 HTML
// Markdown 内容可以是 存储在数据库中的，也可以是 .md 文件中的
$markdown = <<<EOT
# Hello, Parsedown!

This is **Parsedown**, a better Markdown parser in PHP.

- It's easy to use
- It's fast
- It supports [GitHub flavoured Markdown](https://github.github.com/gfm/)

You can learn more about Parsedown at [parsedown.org](https://parsedown.org).
EOT;

$Parsedown = new Parsedown();
$html = $Parsedown->text($markdown);

echo $html;

CommonMark


CommonMark 是一个高度可扩展且现代的 Markdown 库，由 Colin O’Dell 和多位贡献者编写。自 Laravel 6 起，其成为首选的 Markdown 处理器，并在 GitHub 上获得了超过 1,700 个 star，在 Packagist 上报告了超过 3100 万次下载，并有超过 250 个项目将其声明为依赖项。


CommonMark 支持完整的 CommonMark 规范和完整的 GitHub-Flavored Markup 规范，使其成为 PHP 中最完整的 Markdown 库。


CommonMark 库由一个解析器和一个渲染器（HTML）组成，解析器将原始的 Markdown 文本转换为抽象语法树（AST），渲染器将 AST 转换为 HTML。这些组件可以进行交换以适应不同的需求，或者通过插件支持对解析器和渲染器进行更高级别的修改。


# league/commonmark 库还提供了一个命令行工具，提供流畅的 CLI 接口，用于转换 Markdown 文本
