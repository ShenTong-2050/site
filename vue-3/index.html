事件处理

监听事件

语法：通过 v-on 指令 或 @ 符号 来 监听 DOM 事件，并在事件触发时 执行对应的 vue

事件处理器 的 值：

  • 内联事件 处理器：事件 被 触发时 执行的 内联 vue 语句（与 onclick 类似）
  • 方法事件处理器：指向 组件 定义方法 的 属性名 或 路径

内联事件处理

export default {
    data() {
        return {
            count: 0        
        }    
    }
}

<template>
    <button @click="count++">Add 1</button>
    <p>Count is: {{ count }} </p>
</template>

方法事件处理器

export default {
    data() {
        return {
            name : 'stt'        
        }        
    },
    methods: {
        greet(event)     {
            // 方法中的 `this` 指向当前活跃的 组件实例
            alert(`Hello ${this.name} !`)        
            // `event` 是 DOM 原生事件
            if (event) {
                alert(event.target.tagName)            
            }
        }
    }
}

<template>
    <!-- greet 为 methods 中 定义的 方法名 -->
    <button @click="greet">Greet</button>
</template>

方法 与 内联事件 的 判断

方法事件处理器：

  • foo：直接引用一个 名为 foo 的 方法
  • foo.bar：引用一个对象 foo 的属性 bar，该属性是一个方法
  • foo[‘bar’]：引用一个对象 foo 的 属性 bar，该属性是一个方法
<template>
    <!-- 方法事件处理器 -->
    <button v-on:click="foo">Click me</button>
    <button v-on:click="foo.bar">Click me</button>
    <button v-on:click="foo['bar']">Click me</button>
</template>

内联事件 处理器：

  • foo()：调用 foo 方法
  • count++：递增 count 变量
<template>
    <!--内联事件处理器-->
    <button v-on:click="foo()">Click me</button>
    <button v-on:click="count++">Click me</button>
</template>

此处 foo() 与 count++ 都是表达式

内联事件 参数传递

export default {
    warn(message,event) {
        if (event) {
            event.preventDerfault()                    
        }
        alert(message)
    }
}

<template>
  <!-- 使用特殊的 $event 变量 -->
  <button @click="warn('Warn message',$event)">Warnning</button>
  
  <!-- 使用内联箭头函数 -->
  <button @click="(event) => warn('Warn message',event)">Warning</button>
</template>

事件修饰符

  • .stop：终止 事件 的 传播
  • .prevent：阻止默认事件【例如 submit 刷新页面事件】
  • .self：只有 当 event.target 是 元素本身 时 才会触发 事件处理器
  • .capture：在 捕获阶段 触发 事件处理器
  • .once：只会 触发一次 的 事件处理器
  • .passive：不想组织
<script>
export default {
    methods : {
        wrapperFunc() {
            alert('最外层事件')
        },
        test(message,event) {
            alert(message)
            console.log(event.target)
        },multiEvent() {
            alert('multi event')
        }
    }
}
</script>

<template>

    <div class="wrapper" @click="wrapperFunc()">

        <!-- <a v-on:click="test(11,$event)">终止事件向外传递修饰符</a> -->

        <!-- <form v-on:submit.prevent="onSubmit"> --> <!-- 刷新页面 事件 终止 -->
        <!--<form v-on:submit.stop.prevent="onSubmit"> 
            <button type="submit">提交</button>
        </form>-->         <!-- 虽然 刷新页面 事件 终止了，但 提交事件 还是会 向外传递 -->

        <!-- <a @click.stop.prevent="test(22,$event)">修饰符的链式调用</a> --> <!-- 向外传递事件 终止 -->

        <!--<form @submit.prevent>提交</form>-->  <!-- 只有修饰符，事件还是会向外传递，刷新页面事件终止 -->

        <div class="card" @click.self="test(33,$event)">仅当 event.target 是元素本身时才会触发事件</div>

        <!--<div v-on:click.capture="onClickCapture">
            <button v-on:click="onClick">Click me</button>
        </div>-->  <!-- 捕获阶段 触发 事件处理器 -->

        <button class="card" @click.once="test(44,$event)">只会触发一次</button>

    </div>
    
</template>

<style>
.wrapper {
    width: 100%;
    height: 100%;
    background-color: #ccc;
}

.card {
    width: 200px;
    height: 200px;
    background-color: #f00;
}

.btn {
    margin: 20px 100px;
}
</style>

使用链式修饰符需要注意调用顺序，例如：

event.prevent.self：此时 子元素 的 默认行为 也会被 阻止

event.self.prevent：此时 子元素 的 默认行为 不会被 阻止

这两种情况下 self 修饰符 的 作用域 还是 仅限于 当前元素，不会对 子元素 造成影响

<template>
    <!-- 
    @event.prevent.self：应用顺序是 先 prevent 后 self
    此时 div 及其子元素 的 默认行为 都会被阻止，且 只有当 点击div 时 才会调用 test 方法
    
    @event.self.prevent：子元素 的 默认行为 不会被阻止 且 只有点击 div 才会调用 multiEvent 方法
    -->
    <form action="/">
        <div @click.self.prevent="multiEvent">
            DIV Content
            <button class="btn" @submit="multiEvent">提交</button>
        </div>
    </form>
</template>

.passive 与 .prevent 修饰符 是 相互冲突的，所以建议不要同时使用，否则 .prevent 会被忽略

按键 修饰符

export default {
    methods: {
        onSubmit() {
            alert('submit event')        
        },
        onPageDown() {
            alert('page down event')        
        }
    }
}

<form action="/" @submit.prevent="onSubmit">
    <input @keyup.enter="onSubmit" placeholder="Enter 提交" /><br>
    
    <!-- 可以直接使用 `KeyboardEvent.key` 暴露 的 按键名称 作为 修饰符，但 需要转为 kebab-case 形式 -->
    <input type="text" @keydown.page-down="onPageDown" placeholder="PAGE DOWN 按钮">  
    
    <input type="text" @keyup.delete="onSubmit" placeholder="DELETE 按钮" />
</form>

按键别名

  • .tab
  • .delete（捕获 Delete 和 Backspace 两个按键）
  • .esc

系统按键修饰符

  • .ctrl
  • .alt
  • .shift
  • .meta
<template>
    <input type="text" @keyup.alt.enter="clear" placeholder="ALT + ENTER 按钮" /><br>
    <input type="text" @keydown.ctrl.alt="clear" placeholder="CTRL + ALT 按钮" /><br>
</template>

系统按键 修饰符 与 常规按键 不同，当 系统按键 与 keyup 事件 一起使用时 例如： keyup.ctrl，只有当你仍然按住 ctrl 但 松开了另一个按键时被触发。若你单独松开 ctrl 键 将不会触发。

.exact 修饰符

.exact 修饰符 允许 精确控制 触发事件 所需的 系统修饰符 的 组合

<template>
    <!-- 当 按下 Ctrl 时，即便同时按下 Alt 或 Shift 也会触发 -->
    <button @click.ctrl="onClick"> A </button>
    
    <!-- 当 按下  Ctrl 且 未按任何其他键 时 才会触发 -->
    <button @click.ctrl.exact="onCtrlClick">A</button>
    
    <!-- 当没有按下任何系统按键时 触发 -->
    <button @click.exact="onClick">A</button>
</template>

.exact 修饰符 需要与 鼠标 左键 搭配使用

鼠标按键修饰符

  • .left
  • .right
  • .middle

通过 以上 修饰符 将 程序限定为 由 特定鼠标按键 触发

<template>
    <button @click.left="onClick">鼠标左键才会触发</button>
    <button @click.right="onClick">鼠标右键才会触发</button>
    <button @click.middle="onClick">鼠标中间键才会触发</button>
</template>

表单输入绑定

将 表单内容 同步给 vue 中 相应的变量

<template>
    <input :value="text" @input="event => text = event.target.value" />
    
    <!-- v-model 指令 --> 
    <input v-model="text">
</template>
  • input 与 textarea 元素 会 绑定 value 属性 并 侦听 input 事件
  • <input type="checkbox"> 和 <input type="radio"> 会 绑定 checked 属性 并 侦听 change 事件
  • <select> 会绑定 value 属性 并 侦听 change 事件

v-model 会 忽略 任何 表单元素 上 初始的 value、checked 或 selected 属性。

v-model 会将当前绑定的 vue 状态 视为 数据的正确来源，你应该在 vue 中使用 data 选项 来 声明该初始值。

表单输入绑定基本用法

文本

<script>
export default {
    data() {
        return {
             msg : '',     // 初始化
        }    
    }
}
</script>

<template>
    <div>
        <p>输入内容：{{ msg }}</p>
        <input type="text" v-model="msg" placeholder="请输入内容..." />
    </div>
</template>

对于 IME 语言（中文、日文、韩文），v-model 指令不会在 IME 还在输入阶段时 触发更新。

如果想在拼音阶段也触发更新，应使用自己的 input 事件监听器 和 value 绑定，而不要 使用 v-model

多行文本

<script>
export default {
    data() {
        return {
             multiMsg: '', // 初始化
        }    
    }
}
</script>

<template>
    <div>
        <p style="white-space:pre-line;">多行文本内容：{{ multiMsg }}</p>
        <textarea v-model="multiMsg" placeholder="请输入多行文本内容..."></textarea>
    </div>
</template>

选择器

export default {
    data() {
        return {
             hobby:['足球','跑步'],
             hobbyOptions:[
                {value: 1, text: '篮球'},
                {value: 2, text: '足球'},
                {value: 3, text: '乒乓球'},
                {value: 4, text: '跑步'},
                {value: 5, text: '游泳'},
                {value: 6, text: '爬山'},
                {value: 7, text: '骑行'},
                {value: 8, text: '桌球'},
            ],       
        }    
    }
}

<template>
    <div>
        <h2>单个选择器</h2>
        <div>
            <label for="satisfication">满意度评价：{{ satisfication }}</label>
            <div>
                <label for="satisfication">开始选择：</label>&nbsp;
                <select v-model="satisfication">
                    <option value="" disabled>请选择</option> 
                    <option value="A">A</option>
                    <option value="B">B</option>
                    <option value="C">C</option>
                    <option value="D">D</option>
                </select>
            </div>
        </div>
        <h3>多个选择器</h3>
        <div>
            <label for="hobby">爱好：{{ hobby }}</label>
            <div>
                <label for="hobby">开始选择：</label>
                <select v-model="hobby" multiple>  <!-- multiple 是 关键字 -->
                    <template v-for="option in hobbyOptions" :value="option.value" :key="option.value" > 
                        <template v-if="option.value === 2 || option.value === 4 ">
                            <option selected >{{ option.text }}</option>
                        </template>
                        <template v-else>
                            <option >{{ option.text }}</option>
                        </template>
                    </template>
                </select>
            </div>
        </div>
    </div>
</template>

复选框

export default {
    data() {
        return {
            // 绑定值
            bindValueRadio:'',
            bindValueCheckbox:'',
            bindValueSelect:'',        
        }    
    }
}

<template>
    <template>
        <div>
            <div>
                <div>
                    <h2>单选框的值绑定：</h2>
                    <!-- `picked` 在被选择 时 时 字符串 `a`-->
                    <label for="bindValueRadio">给单选框绑定值：{{ bindValueRadio }}</label>&nbsp;
                    <input type="radio" v-model="bindValueRadio" :value="1">小米
                    <input type="radio" v-model="bindValueRadio" :value="2">苹果
                    <input type="radio" v-model="bindValueRadio" :value="3">华为<br>
                </div>
                <br>
                <div>
                    <h2>多选框的值绑定：</h2>
                    <label for="bindValueCheckbox">给多选框绑定值：{{ bindValueCheckbox }}</label>&nbsp;
                    <!--`bindValueCheckbox` 只会为 true 或 false-->
                    <input type="checkbox" v-model="bindValueCheckbox" :true-value="1" :false-value="false"/>Apple
                    <input type="checkbox" v-model="bindValueCheckbox" :true-value="2" :false-value="false"/>Pear
                    <input type="checkbox" v-model="bindValueCheckbox" :true-value="3" :false-value="false"/>Orange<br>
                </div>
                <br>
                <div>
                    <h2>下拉框的值绑定：</h2>
                    <!-- `selected` 在 第一项 被选中时 为 字符串 `abc`-->
                    <label for="bindValueSelect">给下拉框绑定值：{{ bindValueSelect }}</label>&nbsp;
                    <select v-model="bindValueSelect">
                        <option value="abc">ABC</option>
                        <option value="def">DEF</option>
                    </select>
                </div>
            </div>
        </div>
    </template>
</template>

修饰符

  • .lazy：v-model 默认会在每次 input 事件后更新数据（IME除外），可以通过添加 lazy 修饰符 来 改为在 每次 change 事件后 更新数据。
  • .number：自动将用户输入自动转数字【当 input type="number" 时自动启用】（如果无法被 parseFloat() 处理则返回原值）
  • .trim：自动将用户输入两端去掉空格
