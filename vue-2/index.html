计算属性

基础示例

计算属性 是为了 解决 在模板中 书写 比较复杂逻辑 的 问题

例如：

export default {
    data() {
        return {
            name:"Alex"
            books:[
                'book-1',
                'book-2',
                'book-3'            
            ]        
        }    
    },
    computed:{
        publishBooksMessage() {
            return this.books.length > 0 ? 'Yes' : 'No'       
        }    
    }
}

<span>{{ this.books.length > 0 ? 'Yes' : 'No' }}</span> <!--比较复杂--> 
<span>{{ publishBooksMessage }}</span> <!--通过 computed 属性 获取逻辑结果 就 比较简洁了-->

计算属性缓存 与 方法

如果将 上例 中的 publishBooksMessage 写为一个方法也可以实现同样的效果，区别在于 compute 属性 是 仅在 响应式依赖（books） 发生变化的时候 才会更新缓存，而 函数 则是每次都会执行。

可写计算属性

计算属性 computed 默认是 只读 的，但 在 某些 特殊场景中 会 用到 可写 属性，可以通过 getter 和 setter 来创建：

import { debounce } from 'lodash-es'

export default {
    data() {
        return {
            firstName : 'John',
            lastName:'Doe'        
        }    
    },
    created : {
        debounceClick : debounce(this.click,3000)  // 本组件的防抖动实例
    },
    computed: {
        fullName : {
            // getter        
            get() {
                return this.firstName + ' ' + this.lastName            
            },
            // setter
            set(newName) {
                // 解析赋值
                [this.firstName,this.lastName] = newName.Split(' ')
            }
        }    
    },
    mounted() {
        console.log('fullName is '+this.fullName)
    },
    unmounted() {
        this.debounceClick.cancel()
    },
    methods: {
        click() {
            this.fullName = "申 通通"
            console.log('firstName is ' + this.firstName + ' lastName is ' + this.lastName)
            console.log('fullName is ' + this.fullName)
        }
    }
}

<button @click="debounceClick()">修改计算属性的值</button> // 默认计算属性是 只读 的

computed 最佳实践：计算属性 computed 的 get 属性 中 不应有 其他 异步请求 或 更改 DOM 操作，它的职责 是 计算 和 返回结果

绑定 HTML class

绑定对象

<script>
// 动态绑定 对应 js
export default {
    data() {
        return {
            isActive : true,    
            text-danger : true,
            objectClass {
                isActive : true,
                text-danger : false,        
            }
        }
    },
    computed: {
        classObject() {
            return {
                errorClass : !this.isActive && this.text-danger
            }        
        }    
    }    
}
</script>

// 1) 动态绑定
<div class="static" :class="{ active: isActive,text-danger: isActive}"> 
// class 对应结果
<div class="static active text-danger"></div> // 如果 text-danger 为 false 则 class="static active"
// 2) 动态绑定对象
<div :class="objectClass"></div> // 效果参照 1
// 3) 动态绑定 计算结果
<div :class="classObject"></div> // 效果同 1

绑定数组

data() {
    return {
        activeClass : 'active',
        errorClass: 'text-danger'    
    }
}

<div :class="[activeClass,errorClass]"></div>
<div :class="[activeClass ? activeClass : '',errorClass]"> // 数组中使用三元判断
<div :class="[{activeClass : isActive},errorclass]"></div> // 数组中嵌套对象

在组件上使用

当你使用 引入组件 的时候，在 组件上 增加的 class 会动态合并

特殊情况：当你的 引入组件 中 有多个 根元素 时 需要通过 $attrs 属性 来 指定接收 class 的元素，例如：

<!--Mycomponent-->
<div :class="$attrs.class">Hi</div>
<p>text content</p>

<!--使用 引入的 Mycomponent 组件-->
<Mycomponent class="baz" />
// 渲染结果为
<div class="baz">Hi</div>
<p>text content</p>

自动前缀

对于在 :style 中 使用了 需要 浏览器特殊前缀 的 CSS 属性时，Vue 会 自动为他们 加上 响应地前缀。

Vue 在运行时会自动检查浏览器 是否支持该属性

样式多值

<div :style="{display: ['-webkit-box','-ms-flexbox','flex'] }"></div>

在这种情况下 Vue 会 渲染 浏览器 支持的 最后一个 值

条件渲染

<h1 v-if="awesome">hello world</h1>
<h1 v-else-if="awsome == 'Y'">yes</h1>
<h1 v-else>not good</h1>

<button @click="awesome = !awesome">Toggle</button>

template 之 v-if

根据判断结果显示 区块元素，渲染结果并不包含 <template> 元素

<template v-if="ok">
    <h1>Title</h1>
    <p>Paragraph 1</p>
</template>

template 之 v-show

与 template 不同的 是 v-show 会在 DOM 中保留该元素，仅仅是 切换了元素的 display 属性

<h1 v-show="ok"> hello! </h1>

v-show 与 v-if 的区别

  • v-if 的真实性：v-if 区块内 的 事件监听器 和 子组件 都会被 销毁 和 重建
  • v-if 的惰性：初次渲染条件为 false 不会做任何事，为 true 时才被渲染

总的来说，v-if 有更高的切换开销、v-show 有更高的初始渲染开销。

因此 如果需要频繁切换 推荐使用 v-show，如果在运行时绑定条件很少改变，则 v-if 会更合适。

不推荐同时使用 v-if 与 v-for，虽然 v-if 的 优先级 大于 v-for

v-for 与 v-if

同时 在一个元素上 使用 v-for 与 v-if 是不推荐的，v-if 的优先级比 v-for 的 更高，所以 v-if 无法使用 v-for 作用域内的变量

除非 将 v-for 与 template 进行绑定 且 v-if 位于 template 元素内部 可以解决这个问题

v-for 之 key 的使用

总的来说， vue 在 渲染元素列表时 采用的是 “就地更新” 策略，而 key 属性 是 提示 vue 每一个节点的唯一标识，这样在渲染的时候能最大限度的提高效率。

关于什么是 就地更新 策略：

关于 key 的特征：

  • key 的值 为 字符串 或 number 类型
  • key 的值 在 同一 父元素下 的 子元素 必须是 唯一的【重复的 key 会导致渲染异常】

<!-- v-for 与 template 使用的 入口文件  ---->>
<script>

import Vfortemplate from './Vfortemplate.vue'


export default {
    components : {
        Vfortemplate
    },
    data() {
        return {
            newFruitName : '',
            fruits : [
                {
                    id : 1,
                    name : '苹果'
                },
                {
                    id : 2,
                    name : '香蕉'
                },
                {
                    id : 3,
                    name : '橙子'
                }
            ],
            nextFruit : 4,
        }
    },
    methods : {
        addFruit() {
            this.fruits.push({
                id : this.nextFruit++,
                name : this.newFruitName
            })
            this.newFruitName = ''
        }
    }
}
</script>

<template>
    <form v-on:submit.prevent="addFruit">
        <label for="new-fruit">添加水果:</label>
        <input id="new-fruit" v-model="newFruitName" placeholder="例如:香蕉..."/>
        <button>确定</button>
    </form>

    <ul>
        <Vfortemplate 
        v-for="(fruit , index) in fruits"
        :key="fruit.id"
        :title="fruit.name"
        @remove="fruits.splice(index,1)"
        ></Vfortemplate>
    </ul>

</template>

components/fruit/FruitTemplate.vue

<script>
export default {
    props : ['title'], // 获取 父组件 传递过来的 属性
    emits : ['remove']
}
</script>

<template>
<div>
    <li>
        {{ title }}
        <button @click="$emit('remove')">Remove</button> <!---点击 触发 remove 事件-->
    </li>

</template>

关于父子组件的通信方式：

  • 父组件 的 属性 通过 props 传递给子组件
  • 父组件定义的 事件 通过 emit 关键字 传递

数组变化侦测

变更方法 与 非变更方法 的 区别：

  • 变更方法：使用 这些函数 push()、pop()、shift()、unshift()、splice()、sort()、reverse() 修改数组时 Vue 会 监听到这些变化 并 更新相对应的视图。
  • 非变更方法：使用 filter()、concat()、slice()、map()、reduce()、reduceRight()、some()、every() 这些函数时 不会修改 原数组，而是返回一个 新数组 或 某种结果，由于 没有修改 原数组，所以相对应的试图 也不会做更新的操作。
<script>
export default {
    data() {
        return {
            list: [
                {
                    id:1,
                    name:'张三',
                    age:18
                },
                {
                    id:2,
                    name:'李四',
                    age:20
                },
                {
                    id:3,
                    name:'王五',
                    age:22
                }
            ],
            newList: [],
        }
    },
    methods: {
        test() {
            // 会更新视图
            // this.list.push({
            //     id:4,
            //     name:'赵六',
            //     age:24
            // })

            // 不会更新视图
            this.newList = this.list.slice(0,2)
        }
    }
}
</script>

<template>
    <div>
        <ul>
            <li v-for="item in list" :key="item.id">{{item.name}}</li>
        </ul>
    </div>
    <button @click="test">测试修改数组操作</button>
</template>

数组过滤与排序

过滤

export default {
    data() {
        return {
            numbers: [1,2,3,4,5]        
        }    
    },
    computed: {
        getEvenNumbers() {
            return this.numbers.filter(n => n % 2 === 0)        
        }
    }
}

<li v-for="n in numbers"> {{ n }} </li>  <!--对于 多维数组 可以通过循环嵌套 计算 输出-->

排序

<script>
export default {
    data() {
        return {
            rank : [55,76,45,33,1,33,88],       
        }    
    },
    computed : {
        getSortResult() {
            console.log('原始 rank 数组: '+ this.rank)
            // return this.rank.sort((a,b) => a - b)
            return [...this.rank].sort((a,b) => a - b)
        }
    },
    mounted() {
         console.log('组件加载完成后的 rank 数组 : ' + this.rank)    
    }
}
</script>

<template>
    <div> 
        <ul>
            <li v-for="n in getSortResult"> {{ n }} </li>
        </ul>
    </div>
</template>

在 计算属性中 使用 reverse() 和 sort() 函数需要注意 会 导致 原始数组 发生变更，对应的视图 也会 发生改变

推荐通过 数组的副本 来 调用这些方法
